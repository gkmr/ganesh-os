Ganesh OS - how a day comes together

One rule keeps it sane: every field has exactly one owner. Priority is set by triage, dates by the sweeps, new opportunities by the scan, surfacing by the brief, lifecycle by the reply processor.

Where the master views live

  • Today, ranked, cross-domain: the brief's MIT + top three (chat + phone calendar event). Not the raw "due today" list.
  • Tomorrow, time-ordered: the tomorrow-plan file.
  • The eternal pipeline and the other lists: the two decision canvases.

The fences (one owner per field)

  • Priority → the two triage agents only.
  • Dates / dedupe / conflicts / auto-park → the sweeps.
  • New roles → an intake agent (tier-1 creates only).
  • Lifecycle → the reply processor, from explicit decisions.
  • Deletion → never automatic; confirmation-gated.
